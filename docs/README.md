---

## Key documents

### FLOW.md
The canonical spec for the redesigned flow. Step boundaries are **locked** — the prototype must implement exactly these steps.

### DESIGN-SYSTEM.md
Token values, component patterns, and voice guidelines extracted from the real ablefy authenticated UI. This is the source of truth for all visual decisions in the prototype.

### DECISIONS.md
Every significant design or engineering decision made during this project, date-stamped with rationale. Read this before changing anything structural. Append, never edit previous entries.

### SUPPORTING-MATERIALS.md
Mermaid diagrams of both Option A (targeted fix) and Option B (AI course creator). Rendered version at `docs/preview.html`.
